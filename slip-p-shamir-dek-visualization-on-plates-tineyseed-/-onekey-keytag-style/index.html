SLIP-P - Shamir DEK Visualization on Plates TineySeed / OneKey KeyTag - style

December 24, 2025 in projects

SLIP-P is a Python tool that allows visualizing Shamir-split DEKs on plates in a Tinyseed / OneKey KeyTag - style layout. It supports 128, 192, and 256-bit DEKs with dynamic checksum.

The idea started from traditional mnemonic splitting but was optimized to fit binary shares on physical plates for secure offline storage. Each DEK can be split into multiple shares and any threshold of shares can be used to recover the original DEK.

The plates show each bit as a visual mark, grouped in columns representing powers of two and rows for easy human reading. Dynamic checksums ensure integrity during recovery.

Features

  • Support for 128, 192, 256-bit DEKs
  • Threshold-based Shamir splitting (2-of-3 or configurable)
  • Plate visualization in Markdown-friendly format
  • Dynamic checksum verification
  • Easy recovery from any sufficient set of shares

Usage

from slip_plate.main import main

main()

Example usage

Run the example script with a specific DEK size:

python -m examples.example_run --dek 256

Security Model

  • Plates are generated once and stored offline
  • DEKs are reconstructed from a quorum of shares during encryption or decryption
  • Nonces are unique per encryption and embedded in the ciphertext
  • No DEK is stored digitally outside memory during use
