SLIP-P - Shamir DEK Visualization on Plates TineySeed / OneKey KeyTag - style

December 24, 2025 in projects

SLIP-P is a small, open-source demo that helps you split a data-encryption-key (DEK) using Shamir secret sharing and render the resulting shares as human-friendly plates (OneKey/Tinyseed-style). It is intended as a practical exploration and visual aid for securely storing key shares offline.

Why this is useful

  • If you want to split a master key into multiple physical copies (plates) such that any threshold of them can recover the key, SLIP-P shows a simple way to do it and to verify shares with a short checksum.
  • The plate output is optimized for printing or copying to offline media.

Try it (quick start)

  1. Clone the repo:

    git clone https://github.com/avra911/slip-plate.git
cd slip-plate

  2. Create a virtual environment and install dependencies:

    python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt pytest
pip install -e .

  3. Run the demo CLI (prints plates to your terminal):

    python -m slip_plate.cli --dek-size 256 --parts 3 --threshold 2

  4. Or run the example script:

    python -m examples.example_run --dek-size 128 --parts 3 --threshold 2

What you’ll see

  • An encrypted sample blob (DEK used to encrypt a short message).
  • The Shamir shares rendered as plates with a small checksum highlighted - suitable for printing or manual transcription.

Testing and development

  • Run the unit tests with pytest -q.
  • The project contains small modules for checksum handling, plate rendering, AES-GCM helpers and simple Shamir wrappers - easy to read and extend.

Repository

  • The full source and examples are available on GitHub: https://github.com/avra911/slip-plate.

Notes and caveats

  • This is a demo / research tool - treat outputs as examples and review the code and cryptographic choices before using in production.
  • Keep printed plates physically secure; the checksum helps detect transcription errors but is not a replacement for proper key management.

Contributions welcome - open a PR or an issue on the repo if you have improvements or questions.
